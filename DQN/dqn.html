dqn¶

class appeal_rl.dqn.dqn.DQN(env, args)[source]¶

The core of the reinforcement learning is our modified Deep Q-Network with multiple heads (in our case 7 heads); one for each taxonomy level (6 levels) and one for VARK prediction. Total loss is the combined weighted loss from each head. This is a direct implementation of a multi-task learning architecture with a shared encoder.

decode_action(vark_action, taxonomies_action)[source]¶

This function decodes the action taken. For example: it converts a VARK of 4 to a VARK vector of [0100].

get_action_value(state, vark_action, taxonomies_action, network)[source]¶

This function gets the current policy action value in a certain state using the policy network.

get_ep_greedy_action(state, network)[source]¶

This function uses a greedy policy to sample an action. It either gets the action with the maximum Q-value or selects a random action with probability (epsilon)

get_max_action_value(state, network)[source]¶

This function gets the maximum policy action value in a certain state using the policy network.

get_policy_action(state, network)[source]¶

This function gets the current policy action [action with highest Q-Value] in a certain state using the policy network.

train()[source]¶

This function trains the DQN network for a certain number of episodes with a max. unrolling depth. It also computes the statistics of the training process and sends them to Tensorboard for visualization purposes.

Here is the scenario: It takes a step in the environemnt using eps. greedy policy. Then, it adds the obs, rewards, etc. to the replay memory. Then, it samples from the replay memory a batch of a certain size. Finally, it trains the policy network. Note that: after each number of steps, policy network parameters are copied to the target network.

train_single_batch()[source]¶

This function trains the policy network using a single batch sampled from the replay memory.
